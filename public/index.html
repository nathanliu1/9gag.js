9gag.js

9gag.js is an Node.js-based API for 9gag.com. Available in both API server and npm module, it enables third-party applications to access various data from 9gag. Unofficially.

About

Our Web API lets your applications fetch data from 9gag. Based on simple REST principles, our Web API endpoints return data in JSON format about gag data, gags in different sections and user information. However, due to authentication problems, only GET requests are available at the moment.

Getting Started

To run the REST API web service locally:

Pepe - bash $ cd 9gag.js /** Install dependecies **/ $ npm install /** For normal usage **/ $ npm start /** For development usage **/ $ npm run dev /** For testing **/ $ npm test

Port 3000 of localhost is used by default. To run a test request:

Pepe - bash $ curl -X GET "localhost:3000/wtf" {"status":200,"message":"OK","data":["a0jAGDv","avnY4LM","azANoLq","aB1qjQx","aEnryvo","a84Po3e","aK30NYO","aoprqMX","a4jex0d","ae6odpb"],"loadMoreId":"209c3b0762f7ac92e442a40641fff86cd3f11f16","section":"wtf","subSection":"hot"}

To install and use the npm module:

Pepe - bash $ npm install ninegagjs --save
~/example/example.js var ninegag = require('ninegagjs'); var pepe = ninegag.getPost('aLB8VxV', function(response) {
  // Output 200 (Assuming the request is successful)
  console.log(response.status);
  // Output the gag object
  console.log(response.gag);
  // Output the image URL of the gag
  console.log(response.gag.images.large);
});

Features

Easy to Use and Develop - Setup your local server with only a few lines of commands. JSON responses enable ease for reading. Developed on a Express.js layer, along with your favourite JavaScript runtime Node.js, you can start contributing to the project now!

Redis Caching - The web server utilized Redis to constantly cache API responses, reducing the server latency by a large margin.

Infinite Gags - Certain API endpoints provide a loadMoreId to load more items. Simply put your loadMoreId as a query parameter in your URL. Go Fun Yourself with unlimited amount of funny shits!
Example: localhost:3000/wtf?loadMoreId=4cb147723e3d4df803a1074ebba1fa8b9ac275d5 to load more gags in wtf section

Deployment

There are two main branches in this project, master and deploy. master branch is where we develop the API. deploy branch is what being deployed to https://ninegagjs.herokuapp.com/. The deploy branch of 9gag.js is automatically deployed to Heroku for any modification of the branch. We regularly merge master branch into deploy branch for any addition of major features. Continuous Integration is also enabled using Codeship. Each check-in in any branch is verified by an automated test suite, allowing us to detect problems early.
